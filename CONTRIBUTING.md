# Contributing to Midnight dApps

Thank you for your interest in contributing. This repository is a collection of privacy-preserving DApps built on the Midnight Network — Compact smart contracts, React frontends, and end-to-end tutorials. We welcome contributions of all sizes.

## How to contribute

1. **Fork the repository** and create a feature branch (`feat/your-feature` or `fix/your-bug`).
2. **Make your changes** following the existing code style and folder conventions (see [Style.md](./Style.md)).
3. **Open a pull request** with a clear description of what you changed and why. Include:
   - A short summary
   - A test plan (how you verified the change on Preprod)
   - Links to relevant transactions, screenshots, or console logs

## What we accept

- New privacy-preserving DApps built on Midnight (RWA, DeFi, identity, gaming-without-gambling, etc.)
- Improvements to existing contracts, frontends, or tutorials
- Bug fixes and performance optimizations
- Better wallet integration (Lace, 1AM, GSD)
- New analytics scripts, indexer caching layers, or test tooling

## What we do not accept

- **Gambling** or games of chance
- **Lending**, borrowing, or leveraged trading platforms with unbounded risk
- **Prediction markets** or betting mechanisms
- **Inappropriate or harmful content** — hate speech, harassment, violence, adult/explicit material, scams, or deceptive practices

Pull requests that include any of the above will be closed without review.

## Adding a new dApp

If you're adding a fourth (or fifth, sixth…) dApp to this monorepo:

1. Create a top-level folder (e.g. `confidential-yourthing`).
2. Mirror the structure of one of the existing dApps:
   - `contracts/Contract.compact`
   - `src/contracts/` (gitignored — generated by `compact compile`)
   - `src/{pages,components,hooks,lib}/`
   - `package.json`, `vite.config.ts`, `tsconfig.*.json`, `index.html`, `.gitignore`
   - `README.md`, `tutorial.md`, `TWEET.md`
3. Add an entry in the root [README.md](./README.md) under **Projects**.
4. Use the same Compact primitives (commitments + Merkle tree + nullifier) and the same wallet integration helpers so users don't have to relearn each dApp.

## Code style

- Use **TypeScript** for all new frontend code.
- Follow the **existing project structure** — keep wallet helpers in `src/hooks/wallet/` and reusable provider bundles in `src/lib/midnight.ts`.
- Keep code snippets in tutorials **accurate and fact-checked** against the actual source files.
- Use **sentence case** for headings and **active voice** for explanations.
- See [Style.md](./Style.md) for the full guide.

## Compact contracts

- Use `pragma language_version 0.22;` and `import CompactStandardLibrary;`.
- Always seal authority/sponsor public keys: `export sealed ledger authority: Bytes<32>;`.
- Privacy guard pattern: `commitment` → `Merkle tree` → `nullifier`. Don't try to be clever with custom hashes when `persistentHash` works.
- Domain-separate every hash: `pad(32, "yourdapp:purpose:v1")` is non-negotiable.
- Use `HistoricMerkleTree<10, Bytes<32>>` for membership-proof use cases.
